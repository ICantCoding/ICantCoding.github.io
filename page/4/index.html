UPDATE tdb_goods INNER JOIN tdb_goods_cates ON goods_cate = cate_name SET goods_cate = cate_id; #tdb_goods是要更新的表, tdb_goods_cates是参照的表.

连接

内连接, 左外连接, 右外连接. 通常使用ON关键字来设定连接条件, 使用WHERE关键字进行结果集记录的过滤.

table_reference {[INNER | CROSS] JOIN | {LEFT | RIGHT} [OUTER] JOIN} table_reference ON conditional_expr

内连接: 在MySQL中, JOIN, CROSS JOIN 和 INNER JOIN是等价的.

显示左表及右表符合连接条件的记录.

左外连接: LEFT [OUTER] JOIN.

显示左表的全部记录及右表符合连接条件的记录.

右外连接: RIGHT [OUTER] JOIN.

显示右表的全部记录及左表符合连接条件的记录.

无限级分类表设计

自身连接表:

DELETE t1 FROM tdb_goods AS t1 LEFT JOIN (SELECT goods_id, goods_name FROM tdb_goods GROUP BY goods_name HAVING count(goods_name) > 2) AS t2 ON t1.goods_name = t2.goods_name WHERE t1.goods_id > t2.goods_id;

MySQL操作数据表中的记录

  1. INSERT 插入记录
  2. UPDATE 单表更新记录
  3. DELETE 单表删除记录
  4. SELECT 查询语句
  5. 综合举例
INSERT 插入记录

INSERT [INTO] tb_name [(col_name,…)] {VALUES|VALUE} ({expr | DEFAULT},…),(…),…

INSERT [INTO] tb_name SET col_name={expr|DEFAULT},…

INSERT [INTO] tb_name [(col_name,…)] SELECT …

UPDATE 单表更新记录

UPDATE [LOW_PRIORITY][IGNORE] table_reference SET col_name1={expr|DEFAULT}[,col_name2={expr|DEFAULT}]…[WHERE where_condition]

DELETE 单表删除记录

DELETE FROM tb_name [WHERE where_condition]

SELECT 查询语句

别名的使用:

第一种: SELECT id AS userId, username AS uname FROM users; #将查询字段别名化.

第二种:
GROUP BY的使用:

GROUP BY用来分组, 字段的所包含的值中不同值的集合.

SELECT sex FROM users GROUP BY sex;

SELECT sex FROM users GROUP BY 1;#这里的1就是SELECT查询字段中的顺序, 这里1表示sex. 作用同上一句.

HAVING 的使用:

HAVING 是配合GROUP BY一起来使用，HAVING表示筛选条件出来的结果集中，再通过GROUP BY来分组.

注意: HAVING 后边的条件必须是一个聚合函数, 或者条件字段必须是出现在SELECT 后的字段中的字段, HAVING的使用才有效.

聚合函数是指 返回一个结果值的函数.

SELECT sex FROM users GROUP BY 1 HAVING count(id) >= 2; #这里count就是一个聚合函数.

ORDER BY的使用:ORDER BY {col_name | expr | position}[ASC|DESC],…]

举例: SELECT FROM users ORDER BY age DESC, id DESC; #两个条件的排序，先用age排序,再用id排序.

LIMIT 的使用:限制查询结果返回的数量.

LIMIT {[offset,] row_count | row_count OFFSET offset}

举例: SELECT FROM users LIMIT 3,2 #表示从第三条数据偏移后的两条数据.

SELECT * FROM users LIMIT 2 #表示从第0条数据偏移后的两条数据.

综合举例

INSERT INTO test (username) SELECT username FROM users WHERE age >= 30;

MySQL约束以及修改数据表

  4. 表级约束与列级约束

  6. 修改数据表-删除列
  7. 修改数据表-添加约束
  8. 修改数据表-删除约束
  9. 修改数据表-修改列定义
  10. 修改数据表-数据表更名
约束

1.约束保证数据的完整性和一致性.

2.约束分为表级约束和列级约束.

3.约束类型包括:NOT NULL, PRIMARY KEY, UNIQUE KEY, DEFAULT, FOREIGN KEY.

外键约束(FOREIGN KEY)

保持数据一致性,完整性.

实现一对一或一对多关系.

1.父表和子表必须使用相同的存储引擎,而且禁止使用临时表.

2.数据表的存储引擎只能为InnoDB. 在my.ini配置文件中, 修改default-storage-engine=INNODB

3.外键列和参照列必须具有相似的数据类型.其中数字的长度或是否有符号位必须相同;而字符的长度则可以不同.

3.外键列和参照列必须创建索引.如果外键列不存在索引的话MySQL将自动创建索引.

举例:


SHOW INDEXES FROM users\G;查看表的索引列.

外键约束的参照操作

1.CASCADE : 当父表删除或更新,则自动删除或更新子表中匹配的行.

2.SET NULL : 当父表删除或更新行,则设置子表中的外键列为NULL.如果使用该选项,必须保证子表列没有指定NOT NULL.

3.RESTRICT : 拒绝对父表的删除或更新操作.

4.NO ACTION : 标准SQL的关键字,在MySQL中与RESTRICT相同.

举例:

表级约束与列级约束

对一个数据列建立的约束,称为列级约束.

对多个数据列建立的约束,称为表级约束.

列级约束既可以在列定义时声明,也可以在列定义后声明.

表级约束只能在列定义后声明.

修改数据表-添加列

ALTER TABLE tb_name ADD [COLUMN] col_name column_definition [FIRST | AFTER col_name];添加单列.

举例:

ALTER TABLE users1 ADD age TINYINT UNSIGNED NOT NULL DEFAULT 10;

ALTER TABLE tb_name ADD [COLUMN] (col_name1 column_definition,…);添加多列.

修改数据表-删除列

ALTER TABLE tb_name DROP [COLUMN] col_name;删除单列.

ALTER TABLE tb_name DROP [COLUMN] col_name, DROP [COLUMN] col_name2;删除多列.

修改数据表-添加约束

ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] PRIMARY KEY [index_type] (index_col_name, …); 添加主键约束

ALTER TABLE users2 ADD CONSTRAINT PK_users2_id PRIMARY KEY(id);

ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] UNIQUE [INDEX|KEY] [index_name] [index_type] (index_col_name,…);添加唯一约束.

ALTER TABLE users2 ADD UNIQUE (username);

ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] FOREIGN KEY [index_name] (index_col_name,…) REFERENCE reference_tb_name (index_col_name);添加外键约束.

ALTER TABLE users2 ADD FOREIGN KEY (pid) REFERENCES provinces (id);

ALTER TABLE tb_name ALTER [COLUMN] col_name {SET DEFAULT literal | DROP DEFAULT};添加/删除默认约束.

ALTER TABLE users2 ALTER age SET DEFAULT 15;

ALTER TABLE users2 ALTER age DROP DEFAULT;

修改数据表-删除约束

ALTER TABLE tb_name DROP PRIMARY KEY;删除主键约束.

ALTER TABLE tb_name DROP {INDEX|KEY} index_name;删除唯一约束.

ALTER TABLE tb_name DROP FOREIGN KEY fk_symbol;删除外键约束.

修改数据表-修改列定义

ALTER TABLE tb_name MODIFY [COLUMN] col_name column_definition [FIRST|AFTER col_name];修改列定义.

ALTER TABLE tb_name CHANGE [COLUMN] old_col_name new_col_name column_definition [FIRST|AFTER col_name];修改列名称.

修改数据表-数据表更名

ALTER TABLE tb_name RENAME [TO|AS] new_tb_name;

RENAME TABLE old_tb_name TO new_tb_name [,old_tb_name2 TO new_tb_name2]…

MySQL数据表操作

  1. 打开数据库
  2. 显示当前用户打开的数据库

  4. 查看数据库中的数据表
  5. 查看数据表结构

  12. MySQL默认约束(DEFAULT)
打开数据库

USE 数据库名称;

显示当前用户打开的数据库

SELECT DATABASE();

创建数据表

CREATE TABLE [IF NOT EXISTS] table_name (column_name data_type,…)
举例:

查看数据库中的数据表

SHOW TABLES [FROM db_name] [LIKE ‘pattern’ | WHERE expr];

查看数据表结构

SHOW COLUMNS FROM tb_name;

记录的插入

INSERT [INTO] tb_name [(col_name,…)] VALUES(val,…);

举例:

INSERT INTO tb VALUES(‘TOM’,25,7843.25);

INSERT INTO tb (username,salary) VALUES(‘John’,4500.33);

记录的查询

SELECT * FROM tb;

MySQL空值(NULL)与非空值(NOT NULL)

NULL, 字段值可以为空.

NOT NULL, 字段值禁止为空.

举例:

CREATE TABLE IF NOT EXISTS tb (username VARCHAR(20) NOT NULL, age TINYINT);

INSERT INTO tb (NULL,20);#会报错,username不能为空.

MySQL的自动编号(AUTO_INCREMENT)

自动编号, 且必须与主键组合使用.

默认情况下, 起始值为1, 每次的增量为1.

MySQL主键约束(PRIMARY KEY)

1.一张数据表中只能存在一个主键.

2.主键保证记录的唯一性.

3.主键自动为NOT NULL.

MySQL唯一约束(UNIQUE KEY)

1.唯一约束可以保证记录的唯一性.

2.唯一约束的字段可以为空值(NULL).

3.每张数据表可以存在多个唯一约束.

MySQL默认约束(DEFAULT)

1.当插入记录时,如果灭有明确为字段赋值,则自动赋予默认值.
