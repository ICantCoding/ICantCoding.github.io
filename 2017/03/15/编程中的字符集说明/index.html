编程中的字符集说明

  1. 字符集
  2. 中文字符集
  3. 编程中的中文问题
  4. 什么时候需要转换编码？

程序中必不可少的是字符串, 不同国家不同地区拥有不同的字符, 那么如何做到字符间能够识别啦？

字符集

字符集是一个人为的规定, 人们先用一个小册子规定好了”文字字符“与”数字“的对应关系, 其中，每一个字符对应一个数字，这个数字也称为字符的编码.

字符集都是由某组织制定和发布的, 例如ISO/IEC就负责制定字符集.

每个字符集都有一个标准编号, 例如: ASCII字符集的编号为ISO/IEC 646.

我记得我们学习c语言的时候，学到char类型的时候，char和int是可以相互转换的是0~127之间, 那么按照字符集来说，char类型能够对应的字符集也就只能有128个字符了, 因为char是一个字节, 8个位, 这也是为什么char类型是不能容纳除了ASCII字符集以外更多其他字符的原因.

中文字符集

GB2312 中文简体国标码(汉字数: 6763).

GBK 扩展中文GB编码(兼容GB2312).

GB18030 用1-4个字节编码, 容纳的字符数超级多.

GIG5 中文繁体编码.

CJK 中日韩大字符集编码.

Unicode 统一码, 用0-0x10FFFF来映射全球各国的语言文字, 可以容纳0x10FFFF个字符.


GB2312/GBK 双字节编码, 除了ascii码, 每个字节的高位为1, 以此辨别该字符是除ascii码之外的字符.

对于人名, 古汉语等方面出现的罕用字, GB2312不能处理, 这导致了后来GBK及GB18030汉字字符集的出现.

Unicode是国际组织制定的可以容纳世界上所有文字和符号的字符编码方案. 用0-0x10FFFF来映射这些字符,最多可以容纳1114112个字符. 可见Unicode字符集是一个超大的字符集.

那么Unicode用几个字节来表示一个字符啦？

Unicode编码三种方案:

1.UTF-32编码 每个字符用一个int(即4个字节,32位)来表示, 特点:简单, 但是太浪费空间.

2.UTF-16编码 用1~2个short(2个字节,16位)来表示一个字符.

3.UTF-8编码 用1~4个字节来表示一个字符(比较节省空间).

000000-00007F的字符 用 0xxxxxxx来表示

000080-0007FF的字符 用 110xxxxx 10xxxxxx来表示

000800-00FFFF的字符 用 1110xxxx 10xxxxxx 10xxxxxx来表示

100000-10FFFF的字符 用 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx来表示

编程中的中文问题

涉及中文问题时, 我们必须指明所用的字符集和编码！

一遍情况下，我们是使用unicode字符集, 然后采用utf-8编码！这个跟具体的环境有关系，不然也不会出现乱码的情况了. Gbk当然就指的是gbk的字符集和gbk的编码方式了，因为gbk等字符集不像unicode有多种编码集.

什么时候需要转换编码？

你的字符串按GBK编码, 但对方要求按UTF-8编码…..等等情况, 需要转换编码.

char str[] = “中国”; gbk有4个字节的编码, 发送给对方, 但对方只接受utf-8的编码, 这个时候就尴尬了…

注意: 我们在代码中写出来的 汉字, 这些汉字究竟是什么编码方式, 它究竟对应了哪几个字节？ 我们代码中的汉字的编码方式, 是取决于文件本身的编码.

比如: 你在 Centos/Fedora Linux上, 可能默认的字符集存储的是utf-8的. 而在windows中文版中, 默认是按照GBK存储的.
