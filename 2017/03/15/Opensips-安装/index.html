Opensips, 这算是公司通信,调度系统的核心技术了, 我得花时间来研究一下, 技术这个东西, 不常用的话就会忘, 所以还是必须得把学习的东西记录下来. 整个对opensips的学习是基于CentOS 6.5的.

opensips的安装

初学linux, 直接用yum 安装, 不采用源码包安装.

安装步骤一: 安装epel-release

1.yum -y install epel-release #安装epel-release.

安装步骤一: 找opensips的包
  1. 进入opensips的官方网站. www.opensips.org
  2. Downloads 栏目下会提供Packages的下载, 选择好我们服务器操作系统对应的下载地址, 我这里是CentOS6, 对应的是 http://yum.opensips.org 路径, 然后选择2.1stable release(稳定版) X86_64(同时支持32位和64位的操作系统).
  3. 终端运行第2步, 官网提供的yum 地址:

    yum install http://yum.opensips.org/2.1/releases/el/6/x86_64/opensips-yum-release-2.1-3.el6.noarch.rpm
安装步骤二: 开始安装
  1. 安装步骤一完成后, 我们用 yum list | grep opensips* 来查看yum源中对应的opensips相关的rpm包, 从而来选择接下来我们要安装哪一个opensips的包.
  2. 我们可以看到 列出的 rpm包中有一个 “opensips.x86_64” 的包名, 我们接下来就要安装这个rpm包.
  3. yum install opensips.x86_64
  4. 用yum方式来安装, 注意还必须要安装opensips-mysql.x86_64, 这样后边使用opensipsdbctl create创建opensips数据库的时候才有效.

Opensips源码安装

查看源码包中的INSTALL文件, 介绍了如何使用源码包来安装Opensips?

  1. CentOS环境搭建:

    yum install gcc

    yum install mysql mysql-server mysql-devel

    yum install flex

    yum install bison

    yum install ncurses

    yum install ncurses-devel

    yum install pcre* #我当时编译的时候一直报错, 就是因为没有安装这个包.

  2. 下载源码包xx.tar,并解压:

    wget 资源路径

    tar -zxvf 源码包名

  3. 安装:

    进入安装文件夹, 执行make menuconfig命令, 会进入OpenSIPS Main Configuration Menu界面.

    选择Configure Comile Options(编译选项)

    再选择Configure Excluded Modules(配置排除模块)

    在配置排除模块中选中 dialplan 和 db_mysql

    然后返回, 选中Configure Install Prefix, 用于修改安装路径: /usr/local/opensips

    注意一定要Save Changes保存上边的设置

    返回到OpenSIPS Main Configuration Menu界面, 选中Compile And Install OpenSIPS

    然后就是默默的根据我们上边配置的文件, 来编译了.

    编译完成后, 执行make(我在执行make的时候无用, 应该是前面编译的时候已经make过了) , 再执行make install(其实也不用make install) , 那么OpenSIPS就算是安装好了.

  4. 配置:

    编译完成后, OpenSIPS的安装目录就在我们设置的目录/usr/local/opensips目录中.

    安装目录又分几个子目录: etc, lib64, sbin, share

  5. 创建opensips配置中的数据库(前提是已经安装了mysql)

    因为我们自定义的安装文件, 这个文件路径没有在环境变量$PATH中, 所以我们命令实用opensips的时候, 就必须要带命令的绝对路径了.

    /usr/local/opensips/sbin/opensipsdbctl create

6 创建好数据库后, 添加用户

/usr/local/opensips/sbin/opensipsctl add 1001 1001 #添加用户1001, 密码1001 , 将1001用户添加到opensips数据库中的subscriber表中.

7 启动opensips

/usr/local/opensips/sbin/opensipsctl start/restart/stop

8 检查opensips的配置文件语法是否错误: opensips -c

9 使用opensips, 我们相当有必要去配置日志文件. 因为以后任何差错信息, 我们都会到日志文件中去查看. 下边就讲一下如何配置日志文件:

我们的目标是把日志输出重定向到 /var/log/opensips.log 文件下.

9.0 vim /usr/local/opensips/etc/opensips/opensips.cfg 中添加 log_facility=LOG_LOCAL0 #这里的LOG_LOCAL0其实是和后边我们在rsyslog.conf设置的local0.是映射的.

9.1 touch /var/log/opensips.log .

9.2 cd /etc/rsyslog.conf .

9.3 在rsyslog.conf 中添加 local0. -/var/log/opensips.log .

9.4 重启rsyslog /etc/init.d/rsyslog restart

9.5 重启我们的opensips, ./opensipsctl restart

9.6 就配置好了.

源码安装遇见的问题

  1. 我在执行make的时候, 编译出现了两个文件找不到:

    which: no xsltproc in $PATH

    which: no lynx in $PATH

    答: yum install libxslt 解决了没有xsltproc的问题; yum install lynx 解决了灭有lynx的问题.

  3. 因为我是在mac的虚拟机安装的CentOS, 虚拟机网络必须选择桥接模式, 好让我们的mac和CentOS能够在同一网段.

  4. 在开启opensips的时候, 我还多次忘记开启msyql服务, 和ssh服务了.

  5. 关机重启后, 启动opensips发现 启动不了opensips了, 给出如下的错误:

    ERROR: PID file /var/run/opensips.pid does not exist – OpenSIPS start failed

    答: 这个是因为关机重启后, 我安装opensips的服务器的ip地址发生了改变, 我必须要再去更改/etc/opensips下配置文件中涉及到的ip地址信息, 更改到现在服务器的ip地址, 然后重新启动就可以了.

    或者直接永久的关闭防火墙: chkconfig iptables off
