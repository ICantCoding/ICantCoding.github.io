gcc的使用

  1. 编译过程中的gcc会用到的命令
  • 警告选项
  • 优化选项
  • 链接器选项
  • 其他选项
  • 统计程序的运行时间

    • gcc 是linux下标准的编译器, 其中gcc编译涉及到很多命令, 这篇文章, 我会一边学习gcc, 一边记录自己学习过程中的一些技巧和命令

    1. gcc main.c -o main #这个只是简单的编译一个c文件, 不涉及到链接到其他库文件, 生成一个main可执行文件
    2. gcc -g -Wall main.c -o main -L. -lssl #这个是编译main.c文件, 在编译的过程中会提示警告信息和错误信息, 并链接到openssl这个库文件.
    3. gcc -c main.c #这个是将main.c编译成为 main.o目标文件
    4. ar cr libylauthorization.a ylauthorization.o #这个是将ylauthorization.o目标文件 制作成libylauthorization静态库文件.
      4.1 ar cr libylauthorization.a ylauthorization.o libudev.o libudev-enumerate.o #这个表示将多个目标文件做成一个静态库
    5. gcc main.c module.c -o exe #编译main.c和module.c然后生成可执行文件ext, ./exe 是可执行的程序了, 用于编译链接模块化的c程序
    编译过程中的gcc会用到的命令
    1. (预处理) gcc -E main.c -o main.i #预处理命令, 处理那些源文件和头文件中以#开头的命令, 生成.i文件, -E表示只进行预编译

      预处理后生成 预处理后的.c文件
    2. (编译) gcc -S main.i -o main.s 或者 gcc -s main.c -o main.s #编译处理命令, 把预处理完的文件进行一系列的词法分析, 语法分析, 语义分析以及优化后生成相应的汇编代码文件, .s文件.

      编译后生成汇编文件 .s
    3. (汇编) gcc -c main.s -o main.o #汇编，把.s文件, 经过称为汇编语言.

      汇编生成可执行文件 .o
    4. (链接) gcc main.o -o main #将汇编文件, 链接成可执行文件.

      链接后生成可执行文件.
    5. 汇编, 汇编过程就是将汇编代码转换成可以执行的机器指令. 大部分汇编语句对应一条机器指令, 有的汇编语句对应多条机器指令. 汇编的结果是产生目标文件.obj , 目标文件已经是二进制文件, 与可执行的文件组织形式类似, 只是有些函数和全局变量的地址还没有找到, 程序不能执行
    6. 链接, 链接的作用就是将汇编生成的.obj文件, 让.obj文件找到函数, 全局变量的地址, 将所有的目标文件组织成一个可以执行的二进制文件.
    警告选项

    gcc可以通过使用不同的选项来指定编译过程中会提示的警告信息

    -Wall #启用所有的警告信息

    -Werror #在发生警告时取消编译操作, 即将警告看成是错误

    -w #禁用所有警告信息

    -Wformat

    -Wparentheses

    -Wall -Wno-unused #开启所有警告,除了unused警告

    优化选项

    gcc具有优化代码的功能

    -O0 #不进行优化

    -O 或 -O1 #进行基本的优化

    -O2

    -O3

    -Os #生成最小的可执行文件, 主要用在嵌入式领域

    也可以尝试自己添加要优化的具体选项, 例如:

    -finline-functions #允许编译器将一些简单的函数在其调用处展开

    -funswitch-loops #将循环体中值不变的变量移动到循环体之外

    举例:

    gcc -O2 -funswitch-loops example.c -o example #对example.c编译时采用O2优化策略,并添加-funswitch-loops优化选项 .

    还有一个比较重要的优化选项 -march ,它表示为特定的CPU类型编译二进制代码. 进而取得最佳的效果.

    gcc –march= example.c -o example #CUP类型,比如 pentium4, athlon64

    链接器选项

    gcc编译器提供的连接器也提供多个选项:

    -Idirectory #向gcc的头文件搜索路径中添加新的目录

    -Ldirectory #向gcc的库文件搜索路径中添加新的目录

    -llibrary #提示连接程序在创建可执行文件时包含指定的库文件

    -static #强制使用静态连接库

    -shared #生成动态库文件

    gcc example.c -o example -I/home/xxx/include

    gcc example.c -o example -L/home/xxx/lib

    Linux系统中头文件的默认包含路径可以通过环境变量C_INCLUDE_PATH来设定, 库文件的默认搜索路径可以通过环境变量LIBRARY_PATH来设定, 程序运行时加载动态库的查找路径可以通过环境变量LD_LIBRARY_PATH来设定. export LD_LIBRARY_PATH=/home/shange/LearnLinuxProgram/Lib/ 注意要export把变量做成环境变量. 默认先查找的动态库路径为/lib/ , /usr/lib

    显示连接库 : gcc example.c -o example /home/shange/libxx.so

    快捷选项 : gcc example.c -o example -lxx

    gcc编译器在默认情况下使用动态库, 但是如果使用了-static选项, 连接器将忽略动态库.

    gcc example.c -o example -static -lm #这里强制使用静态库

    gcc使用-shared选项来创建动态库

    gcc -shared -fPIC -o libhello.so hello.c #将hello.c文件生成libhello.so文件, 注意这里跟的是.c文件, 之前我用.o文件来做动态库,是不可以的.

    gcc example.c -o example ./libhello.so #使用当前目录下的libhello.so库文件,来生成example可执行文件

    为动态库.so在编译连接的时候，指定路径有三种方式:

    1. export LD_LIBRARY_PATH=.so路径

      export LD_LIBRARY_PATH=/home/shange/LearnLinuxProgram/Lib
    2. -Wl,-rpath,.so路径 #-Wl,表示后面的参数将传给link程序ld（因为gcc可能会自动调用ld）。这里通过gcc 的参数”-Wl,-rpath,”指定.

      gcc main.c -o main -I../ -Wl,-rpath,. -lpone -lptwo
    3. 在/etc/ld.so.conf配置文件中添加 动态库的路径, 保存, 然后在命令行执行 ldconfig 重新加载该配置文件. 虽有gcc 编译, 就可以直接跟.so动态库名称了

      -L. 与 -Wl,-rpath的区别？

      注意: error while loading shared libraries : libabc.so : cannot open shared object file : No such file or directory #这个错误是因为程序运行时没有在系统默认的查找路径中找到动态库文件, 我们需要将libabc.so所在的目录加到环境变量LD_LIBRARY_PATH之中.
    其他选项

    -x language #指定输入文件的编程语言

    -v #显示编译器的版本号

    -g #获得有关调试程序的详细信息

    -ansi #支持符合ANSI标准的C程序

    统计程序的运行时间

    命令行输入 : time ./example

    time命令的输出结果分为三部分:

    real : 程序的总执行时间, 包括进度的调度, 切换等时间

    user : 用户进程执行的时间

    sys : 内核执行的时间
