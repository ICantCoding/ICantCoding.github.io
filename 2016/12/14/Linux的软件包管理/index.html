Linux下的软件, 我们必须要学会安装!不是太难！加油吧！

软件包分类
  1. 1源码包, 没有编译, 高级语言包！


    • 开源


    • 可以自由选择所需的功能


    • 软件是编译安装, 所以更加适合自己的系统, 更加稳定也效率更高.


    • 卸载方便

  2. 2二进制包(RPM包, 系统默认包), 已经经过编译的, 机器语言包!


    • 包管理系统简单, 只通过几个命令就可以实现包的安装, 升级, 查询,和卸载.


    • 安装速度比源码包安装快得多.

RPM包命名原则

httpd-2.2.15-15.el6.centos.1.i686.rpm

httpd 软件包名

2.2.15 软件版本

15 软件发布的次数

el6.centos 适合的Linux平台

i686 适合的硬件平台(还有x64, noarch)

rpm rpm包扩展名

注意: 包名, 包全名的区别. httpd就是包名, httpd-2.2.15-15.el6.centos.1.i686.rpm就是包全名

RPM包依赖性

1.树形依赖 a->b->c (a依赖b, b依赖c, 所以必须先装c, 再装b, 再装c, 卸载的时候就要先卸载a, 再卸载b, 再卸载c).

2.环形依赖 a->b->c->a (a依赖b, b依赖c, c又依赖a, 这就很尴尬呀! 解决办法就是用一条命令同时装a, b, c)

3.模块依赖

模块依赖报错如下:

Failed dependencies: libodbcinst.so.2 is needed by mysql-connector-odbc-5.1.15r1144-7.el6.i686

其中libobdcinst.so.2是软件包库依赖, 它是某一个软件包中的软件, 所以要去安装这个库的软件包, 才能解决这个问题. 我们可以到www.rpmfind.net去找到这个软件库需要安装的包名.

RPM命令

包全名:操作的包是没有安装的软件包时, 使用包全名. 而且要注意路径.

包名:操作已经安装的软件包时, 使用包名. 是搜索/var/lib/rpm/中的数据库,来发现系统中已经安装了哪些rpm包的.

RPM安装: rpm -ivh 包全名

选项 -i (install) 安装

选项 -v (verbose) 显示详细信息

选项 -h (hash) 显示进度

选项 –nodeps 不检测依赖性(一般不用)

RPM包升级: rpm -Uvh 包全名

选项 -U (upgrade) 升级

RPM包卸载: rpm -e 包名

选项 -e (erase) 卸载

查询是否安装: rpm -q 包名

选项 -q (query) 查询

不管我们是用rpm来安装软件包,还是使用yum来安装软件包？我们都会采用rpm查询方式来查询我们的系统是否安装了某一个rpm包.

查询所有安装过得rpm包: rpm -qa

选项 -a (all) 所有

查询安装过软件包详细信息: rpm -qi 包名

选项 -i (information) 信息

注意: rpm包的信息information,其实是在这个rpm包被组建的时候就已经建立好了的, 所以我们不但可以查询已经安装的rpm包的信息,我们还可以查询未安装的rpm包的信息.

查询未安装过得软件包详细信息: rpm -qip 包全名

选项 -p (package)

查询包中文件安装的位置: rpm -ql 包名

选项 -l (list) 列表

选项 -p (package) 查询未安装包信息,需接包全名.

注意: rpm包中文件的安装位置, 其实也在组建这个rpm包的时候就已经确定了的,所以我们不但可以查询已经安装的rpm包的文件安装位置信息, 我们还可以查询未安装的rpm包的文件安装信息.

查询系统文件属于哪个RPM包: rpm -qf 系统文件名

选项 -f (file) 查询系统文件属于哪个软件包

查询软件包的依赖性: rpm -qR 包名

选项 -R (requires) 查询软件包的依赖性

选项 -p (package) 查询未安装包信息, 注意接包全名

RPM包 校验和文件提取

包校验: rpm -V 包名

验证内容中的8个信息的具体内容如下:

S 文件大小是否改变

M 文件的类型或文件的权限(rwx)是否被改变

5 文件MD5校验和是否改变(可以看成文件内容是否改变)

D 设备当中, 代码是否改变

L 文件路径是否改变

U 文件的属主(所有者) 是否改变

G 文件的属组是否改变

T 文件的修改时间是否改变

文件类型如下:

c 配置文件 (config file)

d 普通文件 (documentation)

g “鬼”文件 (ghost file), 很少见, 就是该文件不应该被这个RPM包包含

l 授权文件(license file)

r 描述文件(read me)

RPM包中的文件提取: rpm2cpio 包全名 | cpio -idv .包中文件绝对路径

rpm2cpio 将rpm包转换为cpio格式

cpio 是一个标准工具, 它用于创建软件档案文件和从档案文件中提取文件.

cpio -idv < [文件|设备]

选项 -i 还原copy-in模式

选项 -d 还原时自动新建目录

选项 -v 显示还原过程

举例(如何修复误操作的文件):

查询到/bin/ls文件属于哪个包

rpm -qf /bin/ls

造成ls命令误删除假象

mv /bin/ls /tmp/

提取RPM包中ls命令到当前目录的/bin/ls下

rpm2cpio /mnt/cdrom/Packages/coreutils-8.4-19.el6.i686.rpm | cpio -idv ./bin/ls

把从包中提取出来的ls命令赋值回/bin/目录下, 修复文件丢失.

cp /root/bin/ls /bin/

为Linux配置IP地址

如果在局域网中上网，我们只需要配置ip地址，子网掩码.

如果想上公网, 我们除了ip地址要配置,子网掩码要配置,我们还需要配置网关和DNS.

网络配置, red hat系列的linux可以通过set up
来配置.

eth0 表示是我们电脑的第一块网卡.

DHCP 表示有提供自动分配ip地址的服务器为我们的主机分配ip地址, 如果没有DHCP的服务器, 我们就必须自己来配置ip地址.

静态IP : 192.168.1.156

子网掩码 : 255.255.255.0

默认网关IP : 这个去找网路管理员要, 192.168.1.1

DNS服务器 : 202.106.0.20

进入网卡配置文件 : cd /etc/sysconfig/network-scripts/ifcfg-eth0 把ONBOOT=”no” 更改为 ONBOOT=”yees” #表示启动网卡

配置完成后, 我们需要重新启动网络服务 : service network restart

这样配置ip地址后, 是永久生效的.

网络yum源配置

yum源配置文件放在 /etc/yum.repos.d/的目录下.

[base] #容器名称, 一定要放在[]中.

name #容器说明, 可以自己随便写.

mirrorlist #镜像站点, 这个可以注释掉.

baseurl #我们的yum源服务器的地址. 默认是CentOs官方的yum源服务器, 是可以使用的. 如果你觉得慢可以改成你喜欢的yum源地址.

enabled #此容器是否生效, 如果不写或写成enable=1都是生效, 写成enable=0就是不生效.

gpgcheck #如果是1是指RPM的数字证书生效, 如果是0则不生效.

gpgkey #数字证书的公钥文件保存位置. 不用修改.

yum 命令

yum list #查询源服务器上所有可用的软件包列表.

yum search 关键字 #搜索服务器上所有和关键字相关的包.

yum -y install 包名 #-y表示自动回答yes

yum -y update 包名 #升级

注意:如果yum -y update 不加包名, 就表示升级系统中所有的软件包, 包括linux的内核(这样容易导致linux服务器崩溃), 慎用不加包名的update.

yum -y remove 包名 #卸载

注意: remove 卸载包的时候, a->b->c a依赖b,b依赖c 如果用yum卸载a, 那么在卸载a的时候会把b,c也卸载掉.

yum grouplist #列出所有可用的软件组列表.

yum groupinstall 软件组名 #安装指定软件组, 组名可以由grouplist查询出来.

yum groupremove 软件组名 #卸载指定软件组.

光盘yum源搭建

三个步骤:

1) 挂载光盘
mkdir /mnt

mount /dev/cdrom /mnt/cdrom/

2)让网络yum源失效
mv CentOS-Base.repo CentOS-Base.repo.bak

mv CentOS-Debuginfo.rep CentOS-Debuginfo.repo.bak

mv CentOS-Vault.repo CentOS-Vault.repo.bak

CentOS-Media.repo就是我们光盘硬件做源来使用的repo文件.

3)修改光盘yum源文件
自己配置正确的光盘yum文件.

baseurl=file:///mnt/cdrom #这里是yum源光盘挂载点

enabled=1

RPM包与源码包的安装位置

RPM包安装位置:

/etc/ #配置文件安装目录.

/usr/bin/ #可执行的命令安装目录.

/usr/lib/ #程序所使用的函数库保存位置.

/usr/share/doc/ #基本的软件使用手册保存位置.

/usr/share/man/ #帮助文件保存位置.

源码包安装位置:

安装在指定位置当中, 一般是: /usr/local/软件名/ .

RPM包和源码包安装位置的不同, 带来的影响.

RPM包安装的服务可以使用系统服务(这里的服务是指, 启动, 暂停, 重启)管理命令(service)来管理. 例如: /ect/rc.d/init.d/httpd start 或者 service httpd start

源码包安装的服务则不能被服务管理命令管理, 因为没有安装到默认路径中. 所以只能用绝对路径进行服务的管理. 例如开启apache: /usr/local/apache/bin/apachectl start

源码包的安装

必须要是安装了c语言编译器(gcc).

源代码保存位置: /usr/local/src/ 目录.

软件安装位置: /usr/local/ 目录.

接下来以安装apache源码包为例子来讲解Linux下如何安装源码包:

1) 下载源码包并解压.

2) 进入源码包目录, 一般情况下会看见INSTALL 和 README 两个文件, 前者讲如何安装, 后者讲概述.

3) 根据INSTALL文件的安装步骤, 我们一步一步再来安装.

4) ./configure

./configure 软件配置与检查, 主要功能是:

定义需要的功能选项,

检测系统环境是否符合安装要求,

把定义好的功能选项和检测系统环境的信息都写入Makefile文件中, 用于后续的编辑.

./configure –help 来查看我们可以预定义的一些配置.

查看到我们想预定义的配置后，我们就来配置一下:

./configure –prefix=/usr/local/apache #这里用来配置我们的apache安装目录.

5) ./configure成功之后, 就会生成Mafile文件.

6) make #编译, 就是调用gcc 把源码包翻译成机器语言. make clean可以取消之前make的机器原因.

7) make install #编译安装, 相当于这里才开始真正的安装.

8) PREFIX/bin/apachectl start #启动apache, 其中PREFIX表示之前预设置的时候文件安装路径.
