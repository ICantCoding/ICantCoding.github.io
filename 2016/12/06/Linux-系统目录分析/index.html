今天来了解记录下, Linux系统各目录的作用

存放二进制命令的目录

命令在linux里边都是可执行的二进制文件.命令通常保存在下边四个目录中, 我们分别来看看.

/bin/ : 存放系统命令的目录, 普通用户和超级用户都可以执行。


/sbin/ : 保存和系统环境设置相关的命令, bin前边的s表示super, 表明只有超级用户可以使用这些命令进行系统环境设置。


/usr/bin/ : 注意呀!这里的usr不是user的意思, 在这个目录下, 存放系统命令, 普通用户和超级用户都可以执行(因为bin前面没有s嘛), 这些命令和系统启动无关, 在单用户模式下不能执行!


/usr/sbin/ : 存放根文件系统不必要的系统管理命令.


##### 系统启动分区（目录）

/boot/ : 系统启动目录, 保存系统启动相关的文件, 如内核文件和启动引导程序(grub)文件等

设备保存文件目录

/dev/ : 设备文件保存位置, linux中所有内容都以文件形式保存, 包括硬件（硬盘,软盘,U盘,网卡…等）.那么这个目录就是用来保存所有硬件设备文件的.

配置文件保存位置目录

/etc/ : 配置文件保存位置, 系统内所有采用默认安装方式(rpm安装)的服务的配置文件全部都保存在这个目录当中.

用户家目录(属主目录)

/home/ : home目录下有不同用户的自己家的目录.

系统函数库保存位置

/lib/

lost+found目录

/lost+found/ : 当系统意外崩溃或机器意外关机, 而产生一些文件碎片放在这里. 当系统启动的过程中fsck工具会检查这个目录, 并修复已经损坏的文件系统. 这个目录只在分区中出现, 比如/boot/lost+found/

挂载目录

/media/ : 挂载目录. 系统建议使用来挂载媒体设备的(软盘, 光盘).


/mnt/ : 挂载目录. 这个目录系统建议挂载额外设备, 如U盘, 移动硬盘和其他操作系统的分区.


/misc/ : 挂载目录. 系统建议用来挂载NFS服务的共享目录.

第三方安装软件保存的目录

/opt/ : 第三方安装的软件保存位置. 不过现如今, 大家都喜欢把软件放置到/usr/local/目录下.


/usr/local/ : 作用同/opt/目录

虚拟文件系统

/proc/ : 虚拟文件系统, 该目录中的数据并不保存到硬盘当中, 而是保存到内存当中. 主要保存系统的内核, 进程, 外部设备状态和网络状态灯. 如/proc/cpuinfo就是保存cpu信息的等等…


/sys/ : 虚拟文件系统, 同/proc/都是保存在内存中的. 主要是保存于内核相关信息的.

超级用户家目录

/root/ : 超级用户的家目录.

服务数据目录

/srv/ : 一些系统服务启动之后, 可以在这个目录中保存所需要的数据.

临时目录

/tmp/ : 临时目录, 系统存放临时文件的目录, 该目录下所有用户都可以访问和写入. 每次开机最好都把该目录清空, 不要在该目录下保存重要的数据.

系统软件资源目录

/usr/ : Unix Software Resource, 记住呀, 不是user!! 所以不是存放用户数据, 而是存放系统软件资源的目录.系统安装的软件大多数都保存在这里.

动态数据保存位置

/var/ : 动态数据保存文件. 主要保存缓存, 日志以及软件运行所产生的文件.
